        1.1.1 监听事件 ---  简单逻辑处理
  逻辑代码写在了v-on绑定事件的地方
<button @click="counter += 1">add 1</button> {{ counter }}
        1.1.2 方法事件处理器
v-on 接收一个需要调用的方法名称。
<button @click="greet">add 1</button>    函数可以直接使用event----不加括号
<button @click="greet1($event)">add 1</button>  传入事件对象  ---- 加括号
            greet ( event ) {
                console.log( event.target.tagName )
              },
              greet1 (event) {
                console.log( event.target.tagName )
              }
        1.1.3 内联处理器方法
<button @click="greet1('hi')">say hi</button>  ---- 传参

        1.1.4 事件修饰符  -----  方法只有纯粹的数据逻辑，而不是去处理 DOM 事件细节。
* .stop           給子元素加，可以阻止冒泡
* .prevent        阻止默认事件
* .capture        元素自身触发的事件先在此处处理，然后才交由内部元素进行处理
* .self           給父元素加，可以阻止冒泡
* .once           只触发一次
* .passive        移动端优化使用，不能和prevent结合使用

    3.1 组件
        3.1.1 什么是组件
将一个页面分割成不同的部分，每一个部分都可以看做一块，单独开发这一块，再进行整合就是整个项目，每一块都称之为组件
        3.1.2 使用组件   ----  将组件当做标签来使用
全局注册组件
//定义注册组件
Vue.component('v-header', {
        template: "<div class='header'>我是header</div>"
      });
var vm = new Vue({
        el: "#app",
      data: {
            username:"11111"
         }
      }）
//使用组件
<v-header></v-header>

var vm = new Vue({

        3.1.3 prop
父组件給子组件传值
/**
       * 父组件在调用子组件的地方，添加一个自定义属性title（随便命名-但是要注意语义化）    <v-title title="分类"></v-title>
         属性值为你想要传递的值"分类"
         子组件定义的地方，添加一个props选项，写法为数组，元素为传递过来的属性名   
var Title = {
        props: ['title'],
        template: "#title"
      }
         子组件可以直接使用传递过来的属性名来显示信息
<template id="title">
      <div>
          头部的标题----{{ title }}
      <div>aaa</div>
    </div>
  </template>
       */

        3.1.4 自定义事件
子组件通过事件向父组件传递消息
/**
       * 子组件定义一个方法to-parent，通过某个条件sendData触发此方法 $emit，this.$emit('to-parent', 'k1803')
       在父组件调用子组件的地方，绑定一个子组件传递过来的函数to-parent，<v-title @to-parent = "getData"></v-title>
       事件名称为getData，注意不要加（），
       在父组件处定义一个方法getData(val)，那么val就是子组件传递过来的值
       水井中的水抽到地面，to-parent水管，抽出来的水为'k1803'，
       地面上接水，从水管接，调用子组件处接水<v-title @to-parent = "getData"></v-title>
        getData表示接上来的水
       */
        3.1.5 使用slot分发内容
默认组价使用形如 <v-header></v-header>
         内部写入内容是无效的，类似于给了一个没有开孔的插线板
         想要让内容显示出来，就好比你需要給插线板开孔（2孔、3孔、usb孔）
         在定义组件模板的时候添加slot标签即可完成
         模板内部只写一个slot代表一整个插线板只开了一种孔 --- 单个插槽
         添加多个slot，每一个起一个名字，好比开了几种孔   ----  具名插槽

复用性高
