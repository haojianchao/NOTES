    3.1、官方路由的安装
Vue Router 是 Vue.js 官方的路由管理器。它和 Vue.js 的核心深度集成，让构建单页面应用变得易如反掌。包含的功能有：
  • 嵌套的路由/视图表
  • 模块化的、基于组件的路由配置
  • 路由参数、查询、通配符
  • 基于 Vue.js 过渡系统的视图过渡效果
  • 细粒度的导航控制
  • 带有自动激活的 CSS class 的链接
  • HTML5 历史模式或 hash 模式，在 IE9 中自动降级
  • 自定义的滚动条行为
    3.2、 路由基础
        3.2.1 开始
找到router/index.js,改造  ---- 定义路由
// 1. 引入vue、vue-router,模块化开发必须使用Vue.use(Router)
import Vue from 'vue'
import Router from 'vue-router'

// 2.引入组件---定义组件
import Home from '@/components/home/index.vue'
import Top250 from '@/components/top250/index.vue'
import User from '@/components/user/index.vue'

Vue.use(Router)

// 3.定义路由
const routes = [
{
    path: '/home',
    component: Home
},
{
    path: '/top250',
    component: Top250
},
{
    path: '/user',
    component: User
}
]

// 4、创建路由实例
const router = new Router({
    routes
})

// 5.在main.js入口文件处引入，然后再new Vue实例时添加进去，全局即可访问路由
export default router

import axios from 'axios'

export default {
data () {
return {
name: '',
avatars: '',
alt: ''
}
},
methods: {
getDataSuccess (res) {
// console.log(res.data)
const { data } = res
this.name = data[0].name
this.avatars = data[0].avatars.small
this.alt = data[0].alt
},
getDataFail (err) {
console.log(err)
}
},
mounted () {
const { id } = this.$route.params;
axios.get('http://localhost:8000/api/getCastsRoute?id=' + id)
.then(this.getDataSuccess)
.catch(this.getDataFail)
}
}

        3.2.4 编程式导航
/**
* 结果形如/castsDetail/1231345
* 声明式跳转
* <router-link :to="{name: 'castsDetail', params:{id: item.id} }"></router-link>
* 编程式
* <li @click="goCastsDetail(item.id)></li>
* string: this.$router.push('/castsDetail/' + id)
* object: this.$router.push({name: 'castsDetail', params:{id: id}})
* castsDetail组件获取值
* this.$route.params.id
*/

 {
 path: '/castsDetail/:id',
 name: 'castsDetail',
 component: CastsDetail
 }
/**
* 结果形如/castsDetail?id=12346546
* 声明式跳转
* <router-link :to="{path: '/castsDetail', query:{id: item.id} }"></router-link>
* 编程式
* <li @click="goCastsDetail(item.id)></li>
* string: this.$router.push('/castsDetail?id=' + id)
* object: this.$router.push({path: '/castsDetail', query:{id: id}})
* castsDetail组件获取值
* this.$route.query.id
*/
{
path: '/castsDetail',
component: CastsDetail
}

        3.2.10 动画或者过渡
<transition name="slide"><router-view></router-view></transition>

.slide-enter-active, .slide-leave-active {
transition: opacity .5s;
}
.slide-enter, .slide-leave-to /* .fade-leave-active below version 2.1.8 */ {
opacity: 0;
}

如果将transition标签添加到各个组件，那么可以给每个页面切换时展示不同的效果
給样式表添加scoped属性，表示只在当前组件时生效的

在进入/离开的过渡中，会有 6 个 class 切换。
1、v-enter：定义进入过渡的开始状态。在元素被插入之前生效，在元素被插入之后的下一帧移除。
1. v-enter：定义进入过渡的开始状态。在元素被插入之前生效，在元素被插入之后的下一帧移除。
2. v-enter-active：定义进入过渡生效时的状态。在整个进入过渡的阶段中应用，在元素被插入之前生效，在过渡/动画完成之后移除。这个类可以被用来定义进入过渡的过程时间，延迟和曲线函数。
3. v-enter-to: 2.1.8版及以上 定义进入过渡的结束状态。在元素被插入之后下一帧生效 (与此同时 v-enter 被移除)，在过渡/动画完成之后移除。
4. v-leave: 定义离开过渡的开始状态。在离开过渡被触发时立刻生效，下一帧被移除。
5. v-leave-active：定义离开过渡生效时的状态。在整个离开过渡的阶段中应用，在离开过渡被触发时立刻生效，在过渡/动画完成之后移除。这个类可以被用来定义离开过渡的过程时间，延迟和曲线函数。
6. v-leave-to: 2.1.8版及以上 定义离开过渡的结束状态。在离开过渡被触发之后下一帧生效 (与此同时 v-leave 被删除)，在过渡/动画完成之后移除。
